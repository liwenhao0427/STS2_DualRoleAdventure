# LocalMultiControl 玩家更新与使用说明

> 适用对象：想快速了解这个 Mod 能做什么、怎么用的玩家  
> 统计范围：截至 2026-03-22 11:57（北京时间）的全部更新  
> 排序方式：时间轴逆序（最新在最前）

## 2026-03-22（最新）

### 14:40-15:00｜结束回合切人补强：记录“手动结束意图”避免被瓦库切人干扰
- 修复了“点结束回合后有时不切人”的问题：点击时会记录本次手动结束回合意图，等结束动作回调回来后按该意图执行切人。
- 这样即使中间发生瓦库看门狗触发导致控制ID短暂切换，也不会把“结束回合后的切人”判断误判为无效。
- 切人目标仍按新规则：优先切到下一个可出牌角色，不再盲切下一个序号。

### 14:20-14:40｜回合结束切人规则改为“切到下一个可出牌角色”
- 点击“结束回合”时，先检测全队是否都无牌可出：若是，则保持原规则一次性结束全员回合。
- 若不是“全员无牌”，则不再按角色顺序盲切，改为切到“下一个仍可出牌的角色”，减少切到空手角色导致的无效切换。
- 新增对应日志，便于快速确认本次点击是走“全员结束”还是“切到可出牌角色”。

### 14:00-14:20｜动作队列空引用防崩兜底（防止直接卡死）
- 针对战斗中个别牌（如暗影斗篷等）在本地多控下触发的动作队列空引用，新增双层防崩兜底：即使底层抛出空引用，也会被拦截，避免直接阻塞当前对局。
- 兜底覆盖了动作入队 UI 回调与入队请求两条关键链路，并在命中时自动做一次上下文校正，尽量让后续操作继续可用。
- 该修复优先保证“不中断游戏流程”，后续会继续根据日志缩小触发窗口，逐步从“防崩”收敛到“不触发”。

### 13:05-13:20｜瓦库自动切人补强为“登记后重试”，避免一帧错过
- 修复了“瓦库操作结束后有时不自动切到非瓦库角色”的漏触发：改为先登记自动切换请求，再在后续安全帧重试执行。
- 仍保持“每回合最多一次”规则，不会在同回合反复把你从瓦库视角强制切走。
- 这样即使看门狗结束那一帧正好不安全（例如仍在目标结算窗口），后续帧也会继续尝试，不再直接丢失本回合自动切换机会。

### 12:45-13:05｜瓦库自动切非瓦库改为“每回合最多一次”
- 恢复了“瓦库角色操作完后自动切到可操作非瓦库角色”的流程，并保持安全切换守卫，避免在出牌关键窗口切人。
- 新规则：同一场战斗的同一回合里，这个自动切换最多触发一次。
- 这样你手动切回瓦库角色观察时，不会在同回合被系统再次强制切走；下一回合会重新允许一次自动切换。

### 12:20-12:45｜中和+卡死修复 + 休息区手柄焦点恢复
- 修复了战斗中个别指向牌（如“中和+”）在本地多控下偶发卡死的问题：手动出牌入队前会自动校正到当前卡牌所属角色，避免动作队列归属错位导致无响应。
- 新增“手动出牌临界区”保护：在目标选择到入队的关键窗口里，暂停瓦库自动切人/自动出牌看门狗，减少上下文被抢占引发的异常。
- 修复了休息区切人后的手柄适配：切到下一角色后会自动刷新导航并恢复到可操作选项焦点，手柄可继续直接选项，不需要再切回鼠标。

### 11:40-12:20｜瓦库切人优化 + 进阶难度全开放（本地多角色）
- 文案修正：英文里的 `Wakuu` 已统一更正为 `Vakuu`（例如“全瓦库”对应英文显示 `All Vakuu`）。
- 战斗体验优化：当“所有瓦库角色都无牌可出”时，系统会自动切到下一位仍可操作的非瓦库角色，减少手动切人的频率。
- 进阶难度补齐：本地多角色大厅已开放完整进阶难度档位（`0~10`），并对所有本地角色统一生效，不再受多人进度上限限制。
- 瓦库提示文本已做中英本地化，并统一鼠标/手柄触发文案：会明确提示“瓦库将接管所有角色操作”或“瓦库将接管角色X操作”。  

## 2026-03-21（最新）

### 19:57-20:03｜手柄图标显示与原版一致（仅手柄模式显示）
- 调整了选角界面所有新增手柄提示图标的显示条件：只有检测到“正在使用手柄”时才显示，鼠标键盘模式自动隐藏。
- 该规则覆盖了 `LT + 上下左右` 操作提示，以及瓦库 `Y` / `LT+Y` 的提示图标。
- 图标仅隐藏，不影响对应功能本身；切人、调人数、瓦库快捷键逻辑仍可正常使用。

### 19:45-19:57｜记录输入里程碑 Tag + 新增瓦库手柄快捷键与图标提示
- 已记录输入里程碑 Tag：`snapshot/iter-08.5-lt-input-stable`，用于回溯“LT 组合切人/调人数稳定版本”。
- 选角界面新增瓦库快捷键：手柄按 `Y` 可切换当前编辑角色的瓦库控制；按 `LT + Y` 可一键切换“全体瓦库开/关”。
- 新增手柄提示图标：瓦库单人开关旁显示 `Y`，全体瓦库旁显示 `LT + Y`，方便玩家直观看到可用热键。
- 选角控制区补充了 `LT + 上下左右` 的方向图标提示，并修正了左右方向判定，避免“只触发上下不触发左右”。

### 19:11-19:29｜手柄改为 LT + 右摇杆，并新增 LT 拦截调试日志
- 手柄操作已调整为 `LT + 右摇杆`：选角界面下，右摇杆左右调人数、上下切编辑角色；跑图与战斗中，右摇杆上下切当前控制角色。
- 新增 LT 拦截策略：按住 LT 期间不走原游戏 LT 行为；若仅按 LT 后松开，会在松开时补发一次原 LT 逻辑；若 LT 期间用过右摇杆，则仅执行本地多控逻辑，不触发原逻辑。
- 新增分步日志前缀 `[LT组合]`，可用于定位“是否检测到 LT 按下/释放、是否命中右摇杆、是否成功拦截原逻辑、切人/调人数是否执行成功”。
- 右摇杆连续触发仍按“持续维持方向”生效：保持方向会连发，回中即停。

### 19:29-19:36｜补强 LT 持有期间“全拦截”与右摇杆诊断
- 修复了 LT 持有时仍可能漏入原生手柄逻辑的问题：现在 LT 持有期间会统一拦截控制器原始输入，不再让原摇杆/原 LT 行为抢占。
- 右摇杆读取改为多手柄扫描并输出阈值诊断日志，便于定位“是否有轴值输入、是否越过触发阈值、是否被识别为组合操作”。
- 新增日志 `LT持有期间已全拦截控制器原始输入` 与 `LT按住中，未检测到右摇杆越过阈值`，用于快速区分“拦截链路问题”与“摇杆输入问题”。

### 19:36-19:41｜右摇杆检测补充动作名回退（兼容 Steam 输入映射）
- 修复了“LT 已拦截但右摇杆无触发日志”的场景：右摇杆检测新增动作名回退，不再只依赖轴值。
- 现在会同时读取右摇杆轴值与 `controller_joystick_up/down/left/right` 动作，任一路径命中都可触发切人/调人数。
- 诊断日志补充了轴值与动作态联合输出，便于快速定位是“无输入”还是“输入未达阈值”。

### 19:41-19:45｜LT+摇杆方向判定修正（补齐左右调人数）
- 修复了“上下可切人但左右不触发人数增减”的判定问题：当检测到水平输入时不再被默认纵向优先覆盖。
- 摇杆输入来源扩展到 `joystick_* + dpad_*` 联合识别，兼容不同手柄映射方案。
- 日志文案统一改为“摇杆”并对拦截日志做限频，避免高频刷屏淹没关键触发信息。

### 11:10-11:19｜战利品药水改为全队同步（仅战后奖励）
- 修复了“战后奖励拾取药水只给当前角色、其他角色不同步”的问题。
- 现在仅在战后战利品阶段（非战斗中）触发药水同步：当前角色拿到的药水会镜像发放给其他本地角色。
- 事件、商店、战斗内等非战利品来源不受影响，避免误同步污染流程。
- 新增日志 `战利品药水同步`，可用于核对每个目标角色是否同步成功。

### 10:03-10:10｜工具箱接管策略升级（绕过后台三选一卡死）
- 工具箱接管从“仅瓦库角色”升级为“瓦库角色或后台角色”自动处理：首回合触发时不再打开三选一界面，直接取第一个候选并入手牌。
- 这样可以绕开后台角色不可见选牌界面造成的等待阻塞，避免后续角色首回合空手牌卡住。
- 新增命中日志 `工具箱自动接管已命中` 与结果日志 `工具箱已自动选择首张卡`，便于快速核对修复是否生效。

### 09:56-10:03｜瓦库+工具箱首回合卡死修复（自动选首张）
- 新增工具箱接管逻辑：当角色属于“瓦库角色”且在首回合触发工具箱三选一时，自动选择第一个选项并继续流程，避免卡在选牌动作导致后续无法出牌。
- 该逻辑仅在“本地多控 + 瓦库角色 + 持有低语耳环 + 首回合”条件下生效，其他角色与普通工具箱流程保持原版行为。
- 增加日志 `瓦库工具箱已自动选择首张卡`，便于在 `godot.log` 快速确认修复是否命中。

### 09:47-09:56｜用户名来源层兼容修正（恢复隐藏策略）
- 保留“界面原始长ID文本隐藏”策略，不再把编号显示放到切换按钮上，避免界面噪音。
- 用户名处理从“界面层强改”改为“名字来源层拦截”：统一接管 `PlatformUtil.GetPlayerName`，将本地角色映射为“角色N/Player N”，提升与 DPS 面板、角色详情等依赖玩家名接口的模组兼容性。
- 修复了前一版的时序问题：去掉了本地模式状态依赖，避免角色节点在 `_Ready` 较早阶段写入长ID后无法被替换。

### 09:40-09:47｜用户名显示改为角色编号（不再放在切换按钮上）
- 切换按钮已恢复为纯箭头样式，不再在按钮上显示 `Player N` 文本，避免遮挡与误解。
- 本地多控下接管了玩家名显示来源：战斗侧栏、角色详情页等所有走“玩家名”显示的界面，统一显示为“角色1~角色12”（英文环境 `Player 1~Player 12`）。
- 该调整同时覆盖了长数字 ID 的显示场景（如 `7656...`），现在会替换为对应角色编号名称。

### 09:30-09:40｜瓦库首回合卡住修复 + 角色编号本地化补齐
- 修复了你反馈的“瓦库角色带工具箱时首回合可能跳过、无法执行选卡动作”问题：瓦库自动逻辑改为先确认手里有可打牌，再切换视角执行自动出牌，避免在首回合选卡阶段被提前切走上下文。
- 大厅与战斗中的长数字玩家 ID 已替换为可读编号，统一显示为“角色1~角色12”（英文环境显示 `Player 1~Player 12`）。
- “全瓦库”开关文本已补齐中英本地化，避免英文环境下出现未翻译中文文案。

### 00:35-01:25｜重启房间“直进存档”+ 全链路中英文本地化
- ESC 菜单“重启房间”已改为真正的快速重启：现在会直接读档并进入游戏，不再先停在“多人读档大厅”界面。
- 本地化补齐到关键可见文本：模式卡片标题/描述、本地人数提示、控制角色提示、大厅编辑角色提示、休息区提示、随机角色限制提示等，英文环境会自动显示英文文案。
- 重启失败时会给出明确提示文案，避免无反馈。

### 00:25-00:35｜部署流程改为纯代码模式（仅 dll+json）
- 部署脚本已调整为仅复制 `DualRoleAdventure.dll` 与 `DualRoleAdventure.json`，不再复制 pck。
- 规则文档同步更新：默认不再执行 Godot `--export-pack`，构建成功后直接复制并启动游戏联调。
- `DualRoleAdventure.json` 已更新为 `has_pck=false`，并补全 `description/detail` 说明，明确当前为纯代码实现、无额外素材依赖。

### 00:20-00:35｜ESC 新增“重启房间”快速按钮（含英文文案）
- 在游戏内 `ESC` 暂停菜单新增“重启房间”按钮：本地多人模式下可一键回主菜单并自动进入多人读档，省去手动“保存退出→再点重连”的多步操作。
- 该按钮流程复用了现有多人读档链路，效果与“保存退出后重连”一致，但触发更直接、操作更快。
- 新增英文文案支持：当语言为英文时按钮显示 `Restart Room`，非英文环境显示 `重启房间`，符合游戏菜单双语使用习惯。

## 2026-03-20（最新）

### 22:05-22:30｜v1.05 发版 + 药水栏改为按角色独立
- 已发布 `v1.05`，附件同步为 `DualRoleAdventure.dll`、`DualRoleAdventure.pck`、`DualRoleAdventure.json` 三件套，方便玩家直接覆盖更新。
- 药水规则调整为更符合多人联机：每个角色维护自己的药水位，不再统一塞给 1 号角色，也不再默认额外增加 2 个药水位。
- 现在切换角色时会自动刷新顶部药水栏，显示并可直接使用当前角色自己的药水；战斗前共享顶部栏刷新也会跟随当前控制角色。

### 16:45-16:55｜宝箱重复遗物根因修复（移除二次复制）
- 根据日志确认：5人以上宝箱结算时已通过结果回调覆盖全员发放，额外的手动复制会导致重复发放同名遗物。
- 已移除宝箱特判中的“手动复制遗物”步骤，改为仅依赖结算结果回调统一发放，避免同角色拿到两份同名遗物。
- 结算日志更新为 `已按结算结果发放并结束房间`，便于区分旧链路与新链路。

### 16:35-16:45｜宝箱遗物复制去重修复
- 修复了宝箱房高人数复制遗物时，个别角色可能拿到两份同名遗物的问题。
- 现在复制前会先检查目标角色是否已拥有该遗物：若已拥有则直接跳过该角色复制，避免重复发放。
- 新增日志 `宝箱遗物复制已跳过（目标已拥有）`，便于确认去重逻辑是否命中。

### 16:20-16:35｜藏宝图宝箱结算触发源修复（按全队检测）
- 修复了“由未持有藏宝图的角色进入宝箱房时，其他持图角色不触发藏宝图结算”的问题。
- 现在在宝箱房本地奖励结算后，会对全队做一次藏宝图检测与处理：仅对当前幕仍持有 `SpoilsMap` 的角色执行结算，不会误处理未持有藏宝图的角色。
- 每次命中会输出 `宝箱房间触发藏宝图结算`，整轮会输出 `宝箱房间藏宝图批处理完成`，便于核对是否按预期覆盖所有持图角色。

### 16:00-16:20｜藏宝图全队结算修复 + 瓦库操作自动切视角
- 修复了你反馈的“第二幕宝箱房触发藏宝图时只结算一个角色”问题：现在会把其余本地角色的藏宝图任务一起结算，逐个移除对应藏宝图卡牌，并分别发放 600 金币。
- 增加了同步日志 `藏宝图结算已同步到本地角色`，便于确认每个角色都完成了“移卡 + 给金币”。
- 新增瓦库体验优化：当低语耳环（瓦库）开始自动操作时，会先自动切到对应角色视角，避免“角色在行动但画面停在别人身上”。

### 14:20-14:30｜休息区5+越界改为Finalizer恢复链路
- 确认并修复了“5人以上休息区 `_Ready` 越界导致后续补丁不触发”的链路问题：恢复逻辑从 `_Ready Postfix` 下沉到异常 Finalizer，确保拦截后仍会继续尝试恢复。
- 进入休息区发生越界时，现在会输出结构化日志（玩家数、当前操控角色、房间/本地选项数、加载状态），并在 5 人以上即时弹出“请先按 `R` 或 `]` 切换一次角色”的提示文本。
- 新增自动恢复流程：延迟等待加载稳定后切到 1 号位并分帧刷新选项；恢复成功/失败都会打明确日志，失败时会提示仍需手动切人。
- 模组初始化新增版本生效标记日志：`RestSiteRecoveryV2 loaded (marker=2026-03-20-r1)`，用于快速确认 DLL 是否已更新生效。

## 现在能用到什么（玩家视角）

- 可以在单机里体验本地多角色玩法，并在关键场景切换操控角色。
- 从大厅到战斗、事件、奖励、下一幕，主流程基本可跑通。
- 常见“卡住不动”“按钮不好点”“切人后界面不对”的问题已做多轮修复。
- 按钮和入口交互已经过持续打磨，新玩家上手成本更低。

## 建议你这样体验

1. 先用 2 人开局，熟悉切人按钮和切换节奏。  
2. 在选人界面按需开启“瓦库”勾选：可对单个角色勾选，也可使用“全瓦库”一键开启。  
3. 重点体验：战斗切人、事件选择、领奖励这三段是否顺手。  
4. 若遇到异常，优先记录“在哪一幕、哪个界面、做了什么操作”再反馈，定位会更快。
