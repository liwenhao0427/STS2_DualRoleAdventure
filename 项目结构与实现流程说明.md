# LocalMultiControl 项目结构与实现流程说明（当前版本）

本文档用于快速理解当前“联机转本地多人控制”实现结构。

## 1. 项目目标

- 目标：在单机环境复用联机链路，实现 2~4 人本地多控。
- 原则：优先保证输入路由和状态同步安全，再扩展玩法细节。
- 当前形态：基于 Harmony + baselib 的补丁化实现。

## 2. 目录与职责

- `Scripts/Entry.cs`：Mod 入口，负责补丁注册。
- `Scripts/Runtime/`：本地多控运行时状态、控制权切换与上下文维护。
- `Scripts/Patch/`：按场景/系统划分的 Harmony 补丁（战斗、选人、奖励、事件等）。

## 3. 关键运行时对象

- `LocalSelfCoopContext`：本地多人开关、目标人数、角色切换上下文。
- `LocalMultiSessionState`：会话内玩家顺序与当前控制玩家状态。
- `LocalMultiControlRuntime`：切人行为落地、上下文同步与 UI 刷新。

## 4. 核心流程概览

1. 从联机入口进入“与自己联机”流程。
2. 建立本地回环会话并初始化本地多人队伍。
3. 运行中支持热键/UI 切换控制玩家。
4. 通过补丁维持战斗与非战斗阶段的状态一致性。

## 5. 风险与约束

- 切换控制权必须幂等，避免重复触发导致乱序。
- 高频路径避免重反射和大对象分配。
- 本地镜像状态不得污染底层权威数据。
