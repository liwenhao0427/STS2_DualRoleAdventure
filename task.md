# LocalMultiControl 开发任务单

## 目标（阶段 A）

- 在联机入口新增“与自己联机”。
- 首版固定 2 人本地多控（标准局）。
- 战斗内支持 `[` / `]` 切换当前操控角色。
- 奖励/商店/事件等流程尽量复用多人同步器，资源归属到对应角色。

## 已确认约束

- 首版人数固定 2 人。
- 允许重复角色。
- 共享事件采用自动代投。
- 暂不优先支持 Daily / Custom。

## 任务拆解

### T1 - 本地多控会话骨架

- [ ] 新增本地会话状态管理（2 人固定槽位）。
- [ ] 维护当前操控角色 `CurrentControlledPlayerId`。
- [ ] 提供切换接口：`SwitchNextPlayer()` / `SwitchPrevPlayer()`。

### T2 - 联机入口注入

- [ ] 在联机 Host 菜单注入“与自己联机”按钮。
- [ ] 按钮进入标准局角色选择流程。
- [ ] 默认创建 2 人本地队伍。

### T3 - 本地回环网络适配

- [ ] 提供 `INetGameService` 的本地回环实现（仅本 Mod 使用）。
- [ ] 支持按当前操控角色发送消息（可指定 senderId）。
- [ ] 消息处理保持幂等，避免重复分发。

### T4 - 控制上下文切换

- [ ] 切换角色时同步更新 `LocalContext.NetId`。
- [ ] 确保手牌、结束回合、目标选择跟随当前角色。
- [ ] UI 增加当前操控角色提示。

### T5 - 非战斗流程归属验证

- [ ] 奖励选择归属正确。
- [ ] 商店购买/删牌归属正确。
- [ ] 事件选择（含共享事件自动代投）归属正确。

### T6 - 日志与自检

- [ ] 关键路径日志前缀统一 `[LocalMultiControl]`。
- [ ] 本地构建通过：`dotnet build LocalMultiControl.csproj -c Debug`。
- [ ] 格式检查通过：`dotnet format LocalMultiControl.csproj --verify-no-changes`。

## 开发顺序

1. 先做 T1 + T2，打通入口到开局路径。
2. 再做 T3 + T4，打通战斗内切换闭环。
3. 最后做 T5 + T6，补齐流程与验证。
